# 日记风格化（沈从文/龙应台语感）改写工具

## 这是什么

一个本地 bash+python 脚本（`scripts/diary`），调用本机 Ollama 上的小模型，把口语化、混乱的日记草稿改写成更书面、带一点沈从文（朴素清丽、克制留白）与龙应台（理性、清醒、温和）语感的文字，自动复制到剪贴板，贴进日记本即可。完全本地运行，不联网、不经过任何云端模型。

## 安装到新机器

1. 确认本机已装 [Ollama](https://ollama.com) 并拉取过 `qwen3:8b`：
   ```
   ollama pull qwen3:8b
   ```
2. 把 `scripts/diary` 复制到 `~/.local/bin/diary`（确认 `~/.local/bin` 在 `$PATH` 里），赋予执行权限：
   ```
   cp scripts/diary ~/.local/bin/diary
   chmod +x ~/.local/bin/diary
   ```
3. 新开一个终端窗口（或 `source ~/.zshrc`）即可使用。

## 用法

```
diary                # 提示粘贴原文，Ctrl-D 结束输入
diary 草稿.txt        # 直接传文件
DIARY_MODEL=qwen3-14b:nothink diary   # 换模型（不推荐，见下）
```

输出会打印在终端，并自动 `pbcopy` 到剪贴板。

## 关键踩坑记录（不是理论，是实测踩出来的）

这个功能最大的难点不是"怎么描述风格"，而是**怎么让本地小模型在追求文采的同时不胡编乱造**。踩坑顺序：

1. **第一版**：用一段话描述"沈从文式细节、龙应台式思辨"，让模型自由创作改写。
   结果：模型会编造原文完全没提到的场景——给"梦见外婆"这种抽象描述，自动加上"坐在老屋门槛上、手里捻着豆角"之类的具体动作和道具；给"下雨"加上"邻居阿婆递姜糖"的回忆。**越抽象、越简短的原文，模型越容易脑补出一整套陈词滥调式的乡土场景去填空**（老屋门槛、雨声、稻田、青苔是这个模型的高频幻觉意象，反复出现）。

2. **加"禁止编造"的硬性规则**、区分"比喻"和"编造事实"、举反例——只能压制住一部分。模型仍会把"她总说时间过得快"这种从未发生过的对话/记忆，当成基于"比喻"的合理发挥写出来。**单靠堆规则，对 8B 模型的指令遵从能力来说边际收益很快归零**。

3. **试过用"两阶段：先自由创作，再让模型对照原文核查删编造内容"**——核查这一步本身也会失灵（模型在核查阶段一样会"看不出"自己刚编的内容是编造的）。**不要指望同一个弱模型既能自由生成又能可靠地自我纠错**，这是两种不同难度的任务，发现式核查不能完全代替生成阶段的约束。

4. **最终有效方案：把"自由创作"改成"逐句改写"**——强制模型按原文句子数量逐句输出，不许合并/拆句去补充新内容，每句改写只能基于"这一句"已有的信息，比喻只能用在原文已提及的情绪/时间上。**收窄生成的自由度，比依赖模型自己守约束可靠得多**。代价是文采明显变弱（基本只是把口语理顺、换书面词），但用户的优先级很清楚：**日记首先要求真实，文学性是第二位的**，宁可朴素也不能编造。

5. **加了沈从文《边城》开篇、龙应台《目送》经典段做风格参照（few-shot），而不是只用文字描述风格**——这个改动主要是提升语感真实度，对压制编造没有明显帮助（"逐句改写"的约束本身才是关键），但也没有让幻觉变多，可以保留。

6. **模型选择**：`qwen3-14b:nothink`（社区 abliterated 魔改版）比官方 `qwen3:8b` 更容易胡编、更容易泄露 `<think>` 标签——魔改可能削弱了指令遵从能力。**官方未魔改的小模型，在"必须听指令、不能自由发挥"这类任务上，比同等或更大的魔改模型更可靠**。默认用 `qwen3:8b`，不要切去 14B 魔改版。

7. **`set -u`（bash 严格未定义变量检查）在某次实际使用中触发了一次无法复现的 `unbound variable` 报错**——具体原因没查清楚，但既然这个脚本的逻辑不依赖这层检查来保证正确性，直接去掉 `-u` 换稳定性更值得。**遇到无法复现的 shell 报错，先看这条防御性检查是不是必要的，不必要就关掉，别死磕复现**。

8. **本地模型调用没有任何进度反馈，看起来就像卡死**——尤其是第一次调用某个模型需要先加载到内存/GPU。**任何本地 LLM 调用的脚本都该在发请求前立刻打印一行"正在处理，请稍候"，并且对 URLError/超时/空响应都要给出具体报错文案，而不是无声挂起**。

## 如果要继续改进

- 想要更有文采，可以放宽"逐句改写"的约束，但每放宽一步都要拿"梦见外婆/记不清细节"这类**信息量很少的抽象输入**去测试，这是最容易触发幻觉的输入类型，比"加班淋雨"这种细节丰富的流水账更能暴露问题。
- 真要保留更强的文学性又要保真，下一步该尝试的方向是更大的模型（如果硬件允许），而不是在 8B 模型上继续堆提示词——已经测试过这条路径的边际收益很低。
